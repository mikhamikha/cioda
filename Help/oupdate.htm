Работа с БД: Изменение данных записи в таблице БД
 
Метод oupdate(int, string, string, int)

Модифицирует запись в базе данных, обновляя ее в соответствии с текущим значением тэгов, указанных в Списке Связей (BindList)

Синтаксис

oupdate( connectionId, sXpath, sDoc, suff )

Параметры  
   
connectionId Тип: Целочисленный (32 бита)
  Идентификатор подключения к БД
   
sXpath Тип: Строковый
  XPATH-запрос поиска списка связывания, например, .//BindList[@name="INSERT_SENSOR_READING"]
   
sDoc Тип: Строковый
  Путь до XML-файла, в котором хранится искомый BindList
   
suff Тип: Целочисленный (32 бита)
  Суффикс, добавляемый к имени тэга в BindList
   

Возвращаемое значение

Тип: Integer
0, если успешно, или другое значение, если операцию выполнить не удалось

Примечание

Список связывания (BindList) сопоставляет между собой переменную ввода-вывода Citect и столбец таблицы базы данных.
Метод позволяет выполнить изменение данных в таблице БД (название также указано в BindList) непосредственно из переменных Citect.
Условие может быть задано для одного столбца, соответствие для которого задано елементом where указанного BindList'а
Идентификатор подключения был получен методом oconnecthidden().

Пример

Пример списка связывания (BindList):

    <BindList name="UPDTEST" table="OUT_WBILL_RAW">
      <item where="'lineid#0'" column="LINE_ID" />
      <item name="LI71#0" column="IN_VOLUME" />
      <item name="TI71#0" column="IN_WEIGHT" />
    </BindList>

    #0 - символы форматирования, позволяющие применять один бинд-лист для разного набора тэгов, например, LI7101, LI7102 и т.д.
    Количество знаков от # до конца строки определяет длину суффикса в имени тэга.
    Атрибут name в апострофах ( name="'lineid#0'" ) означает, что значение в выражении тоже будет в апострофах.

Пример на Cicode:

        DllClassCallMethod( hODAC, "oupdate", conId2, ".//BindList[@name=^"UPDTEST^"]", sXML, 4 );
