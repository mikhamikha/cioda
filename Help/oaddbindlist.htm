Работа с БД: Привязка bind-листа к запросу к БД
 
Метод oaddbindlist(int, string, string)

Привязка бинд-листа к сохраненному ранее запросу к БД

Синтаксис

oaddbindlist( stmtId, sXpath, sDoc )

Параметры  
   
stmtId Тип: Целочисленный (32 бита)
  Идентификатор запроса
   
sXpath Тип: Строковый
  XPATH-запрос до бинд-листа, например, .//BindList[@name="TANKINI"]
   
sDoc Тип: Строковый
  Путь до XML-файла, в котором хранится искомый бинд-лист

Возвращаемое значение

Тип: Integer
0, если успешно, или другое значение, если операцию выполнить не удалось

Примечание

Бинд-лист (bind-list) связывает между собой переменную ввода-вывода Citect и столбец таблицы базы данных.
Привязка бинд-листа позволяет выполнить выборку данных из БД непосредственно в значения переменных Citect.
Идентификатор запроса был получен методом osetstatement().

Пример

Пример бинд-листа:

    <BindList name="TANKINI">
      <item name="tankNum#0" column="NUM_STOR" />
      <item name="tankID#0" column="ID" />
      <item name="tankHeight#0" column="BASE_HEIGHT" />
      <item name="tankDeadRest#0" column="DEAD_REST" />
      <item name="tankFUEL#0" column="NAME" />
      <item name="tanksensorID#0" column="SENSOR_ID" />
    </BindList>

    #0 - символы форматирования, позволяющие применять один бинд-лист для разного набора тэгов, например, tankNum01, tankNum02 и т.д.
    Количество знаков от # до конца строки определяет длину суффикса в имени тэга.

Применение см. в oexecute()
