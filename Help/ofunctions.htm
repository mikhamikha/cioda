Описание методов компонента cioda

Этот раздел содержит описание функций для работы с базой данных Oracle и функций логгирования.
Следует помнить о том, что все операции SQL выполняются синхронно.
Управление не возвращается приложению Citect, пока не завершена работа SQL (выполнение опроса, трендов и т. д. приостанавливается).
Поэтому, данные функции рекомендуется выполнять в отдельном потоке, созданном, например, функцией TaskNew()

Все функции SQL, за исключением oconnect(), oconnecthidden(), osetstatement() и ogetrecordnumber(), возвращают код результата операции.
Если код не равен нулю, это значит, что функция не отработала успешно, и следует перейти к другим операциям.

Общий формат функций SQL:

DllClassCallMethod( hODAC, "ofunction", arg1, arg2, ... );

Функция Описание

oaddbindlist

 подключает к запросу, созданному osetstatement(), список связывания
oconnect подключает приложение Citect к базе данных, используя явное задание имени пользователя и его пароля
oconnecthidden подключает приложение Citect к базе данных, используя защищенное задание имени пользователя и его пароля
odisconnect отключает приложение Citect от базы данных
oend освобождает ресурсы, занятые под хранение запроса, параметров и результатов выполнения запроса к БД
oexecute выполняет запрос, подготовленный osetstatement(), oaddbindlist() и osetparameter()
ogetrecord выбирает запись по номеру в таблице результатов, созданную функцией oexecute()
ogetrecordnumber выбирает количество строк из таблицы результатов
oinsert вставляет новую запись в таблицу БД, используя значения тэгов в имеющемся BindList. Параметр BindList определяет, какой из тэгов Citect используется и с каким столбцом БД он связан
opostconnect выполняет дополнительную авторизацию пользователя
osavedbuser сохраняет в конфигурационном файле имя пользователя и в защищенном виде его пароль
osetparameter устанавливает значение именованного параметра для параметризированного запроса
osetstatement загружает в буфер из строки или из конфигурационного файла выражение SQL
oupdate обновляет данные в таблице БД, используя значения тэгов в имеющемся BindList. Параметр BindList определяет, какой из тэгов Citect используется и с каким столбцом БД он связан
initlog инициализация системы логгирования
log создание в лог-файле записи информационнго сообщения
logerr создание в лог-файле записи сообщения об ошибке
